[TOC] ## 一、计算机基础概念 ### 1. 计算机的组成 - 组成（从基本结构上）：运算器、控制器、存储器、输入设备和输出设备 - 计算机的核心部分是 CPU，也就是中央处理器；主要由 运算器、控制器 组成 - 运算器：实现算术运算和逻辑运算的部件 - 控制器：计算机的指挥系统；控制器通过地址访问存储器，从存储器中取出指令，经译码器分析后，根据指令分析结果产生相应的操作控制信号作用于其他部件，使得各部件在控制器控制下有条不紊地协调工作 - 存储器：用来存放所有数据和程序的记忆部件，它的功能是按指定的地址存（写）入或者取（读）出信息；存储器由若干个存储单元组成，每个存储单元都有一个地址，计算机通过地址对存储单元进行读写 - 分类：一类是内存储器，简称内存或主存；另一类是外存储器（辅助存储器），简称外存或辅存 - 单位：一个存储器所包含的字节数称为存储容量，单位有B、KB、MB、GB、TB等 - 输入设备：向计算机中输入信息的设备 - 输出设备：主要有显示器、打印机和绘图仪等 - 工作机制： - 可简述为：没开机的时候，程序存在在硬盘；开机后先运行操作系统，首先进入内存，等待状态开机完成后，操作人员的的指令，指令传给UPC后给出回应，告诉硬盘加载程序。 ### 2. 客户端 - 概念：向服务器索取服务能力的终端；如：手机、平板等 - 通过安装不用的客户端，可以获取不同的服务；常见的客户端软件有：浏览器、QQ、迅雷、Foxmail等 ### 3. 服务端 - 概念：能够提供某种服务的机器（计算机），称为服务端；可理解为：能够提供某种服务的计算机，被称为服务端 - 服务器类型：按照不同的划分标准，服务器可划分： - 按服务类型可分为：文件服务器、数据库服务器 - 按操作系统可分为：Linux服务器、Windows服务器等 - 按应用软件可分为：Apache服务器、Nginx 服务器、IIS服务器、Tomcat服务器、Node服务器 - 服务器软件：使计算机具备提供某种服务能力的应用软件，称为服务器软件；常见的服务其软件如下： - 文件服务器软件：Server-U、FileZilla、VsFTP等 - 数据库服务器软件：Oracle、MySQL、PostgreSQL、MSSQL等 - 邮件服务器软件：Postfix、Sendmail等 - web 服务器软件：Apache、Nginx、IIS、Tomcat、NodeJS等 ## 三、IP 与 域名 ### 1. IP - IP 是 Internet Protocol 的缩写，意思是 “网络之间互连的协议” ，也就是为 计算机网络相互连接进行通信而设计的协议。任何厂家生产的计算机系统，只要遵守 IP协议 就可以与因特网互联互通 - IP 地址：是连接在互联网上的每台计算机分配到的一个32位的地址（二进制）；也是计算机的唯一编号 - 查看IP地址 - mac系统 在终端输入指令 `ifconfig` - window系统 在终端输入指令 `ipconfig` - IPV4 与 IPV6：在windows 7以上系统中，在设置本地IP地址的时候经常会看到同时含有IPV4协议项 与 IPV6协议 - IPV4：版本号为4的IP协议，32位二进制数字，随着IP地址的需求量增大，IPV4地址可能会被用完，便制定了IPV6协议 - IPV6：IPv6是下一版本的互联网协议，128位二进制数字，IP地址量足够大 ### 2. 域名 - IP地址难以记忆，便产生了域名；域名是 IP 地址的 “面具”，也就是网址 ### 3. IP地址 与 域名 之间的关系 - DNS 域名解析：机器间互相只认IP地址，域名与IP地址之间是一一对应的，它们之间的转换工作称为域名解析（域名解析需要由专门的域名解析服务器来完成，整个过程是自动进行的） - 映射关系：一个域名只可以对应一个ip地址，但一个ip地址不一定只对应一个域名 ### 4. 访问网站过程 - 输入网址 ---> DNS服务器上DNS解析 ---> DNS服务器将IP给浏览器 ----> 浏览器通过IP查找本地host文件 --->（没有的话）访问服务器 ---> 服务器返回内容给浏览器 ---> 浏览器进行渲染页面 ### 5. 默认端口号 - 默认HTTP的端口号为80 - HTTPS的端口号为443 ## 四、URL 概述 ### URL - URL 的全称为 `Uniform Resource Locator`，中文叫 统一资源定位符，是互联网上用来标识某一处资源的地址。 - 剖析 `http://www.aspxfans.com:8080/news/index.asp?boardID=5&ID=24618&page=1#name` - 协议部分： `http:` 概念（待更新）；协议后面用 "//" 分割 - 域名部分： `www.aspxfans.com` - 端口部分： `8080` 跟在域名后面的是端口，域名和端口之间使用 ":" 作为分隔符 - 虚拟目录部分： `/news/` 从域名后的第一个 "/" 开始，到最后一个 "/" 为止 - 文件名部分： `index.asp` 从域名后的最后一个 "/" 开始到 "?" 为止 - 锚部分： `name` 从“#”开始到最后 - 参数部分： `boardID=5&ID=24618&page=1` 又称搜索部分、查询部分；从 "?" 开始到 "#" 为止之间的部分为参数部分 ### URL 与 URI 的区别 - URI 全称 `uniform resource identifier`，中文叫 统一资源标识符，用来唯一的标识一个资源 - URL 的全称为 `Uniform Resource Locator`，中文叫 统一资源定位符 ## 五、虚拟主机 - 虚拟主机是使用特殊的软硬技术，把一台真实的物理服务器主机 分割成 多个逻辑储存单元；每一个 逻辑储存单元都没有物理实体，但每一个都能像真实的物理主机一样，在网络上工作，具有独立的 IP、独立的域名、完整的服务器功能 - 一个电脑上可配置：多个虚拟主机 - 虚拟主机集成环境包含：服务器、数据库、运行在服务端的编程语言；常见的集成 软件如下 - mac系统：wamp软件 - window系统：apache软件 ## 六、计算机 体积单位 1TB=1024GB 1GB=1024MB 1MB=1024KB 1KB=1024Byte 注：Byte就是B也就是字节 KB是千字节 MB是兆 GB是千兆 TB是千千兆
